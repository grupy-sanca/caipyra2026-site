Construindo uma Cultura de Code Review
.ical
07/06/2026 14:00–14:45, Palestras - Auditório B

O que você avalia na revisão de um código? Faz isso até nas suas próprias PRs? Vou abordar a importância do code review para a colaboração e passagem de conhecimento com dicas para autores e revisores

Quais conhecimentos prévios são necessários para que seja possível acompanhar bem a sua atividade?:

A palestra é voltada para iniciantes em programação e não é necessário nenhum conhecimento em python. É uma palestra voltada para quem programa em qualquer linguagem ou quem esteja curioso como é feito um code review, mas seria bom ter uma ideia do conceito de contribuição para projetos com PRs.

O que as pessoas que participarem podem esperar aprender na sua atividade?:

Gostaria que as pessoas saíssem da palestra com outra visão sobre o processo de code review, que não é apenas um processo burocrático e de segurança para o projeto que estão trabalhando, mas sim uma oportunidade de aprendizado seja para autores ou revisores.

Além disso, o code review deve ser tratado como uma cultura no grupo ou empresa, com definições e orientações e ser feito com calma e atenção, não apenas para “passar no CI”.

Trarei uma série de dicas para autores e revisores, mas algumas principais que gostaria de deixar fixadas são:

  • fazer PRs pequenas
  • revisar as próprias PRs
  • facilitar o entendimento de quem vai ler aquela PR
  • revise com atenção, calma e tratando como prioridade, pois uma pessoas está “travada” se você não revisar
  • revise os testes
  • enalteça pontos interessantes
